# 模式A：行业周报模板

## ⛔ 收录标准（必须同时满足）

### 检查1：时效性
```
天数差 = 触发日期 - 新闻发布日期

✅ 天数差 ≤ 7 → 可收录
❌ 天数差 > 7 → 禁止收录
```

**示例（触发日期为2026年3月2日）**：
| 新闻发布日期 | 天数差 | 结果 |
|-------------|--------|------|
| 2026-03-02 | 0 | ✅ 收录 |
| 2026-02-23 | 7 | ✅ 收录（边界） |
| 2026-02-22 | 8 | ❌ 禁止 |
| 2026-02-20 | 10 | ❌ 禁止 |
| 2026-01-28 | 33 | ❌ 禁止 |
| 2025-12-01 | 91 | ❌ 禁止 |
| 2022-xx-xx | >1000 | ❌ 禁止 |

### 检查2：行业相关性
```
新闻主体必须属于：设计院、施工单位、建筑企业、房地产、建筑类高校

❌ 禁止收录：
- 互联网公司（程序员、腾讯、阿里、字节）
- 游戏公司（米哈游、网易游戏）
- 金融行业、制造业等非建筑行业
```

**示例**：
| 新闻内容 | 行业 | 结果 |
|---------|------|------|
| 设计院员工猝死 | 设计院 | ✅ 收录 |
| 建筑工人讨薪 | 施工单位 | ✅ 收录 |
| 程序员猝死 | 互联网 | ❌ 禁止 |
| 米哈游员工猝死 | 游戏公司 | ❌ 禁止 |
| 腾讯裁员 | 互联网 | ❌ 禁止 |

---

## 标准输出格式

**标题日期规则**：
- 结束日期 = 触发日期（今天）
- 起始日期 = 触发日期往前数7天
- 共8天

**计算方法**：从触发日期往前数7天，得到起始日期

```markdown
# 一周建筑烂新闻 | MM月DD日 - MM月DD日

> 📊 本周共收录 **N** 条行业负面新闻

**示例**：
- 触发日期3月2日 → 往前数7天是2月23日 → 标题：# 一周建筑烂新闻 | 2月23日 - 3月2日

**验证方法（触发日期3月2日）**：
| 第N天 | 日期 | 状态 |
|:-----:|------|:----:|
| 第0天 | 3月2日 | ✅ 可收录 |
| 第1天 | 3月1日 | ✅ 可收录 |
| 第2天 | 2月28日 | ✅ 可收录 |
| 第3天 | 2月27日 | ✅ 可收录 |
| 第4天 | 2月26日 | ✅ 可收录 |
| 第5天 | 2月25日 | ✅ 可收录 |
| 第6天 | 2月24日 | ✅ 可收录 |
| 第7天 | 2月23日 | ✅ 可收录（边界） |
| 第8天 | 2月22日 | ❌ 禁止 |
| 第10天 | 2月20日 | ❌ 禁止 |
```

# 模式B：单位查询模板

## 触发条件

用户消息以"帮我查询"开头：
```
示例：
- "帮我查询深圳市建筑设计研究总院" → 提取单位名称：深圳市建筑设计研究总院
- "帮我查询中建三局" → 提取单位名称：中建三局
- "帮我查询绿地建设集团" → 提取单位名称：绿地建设集团
```

## 单位名称匹配验证

**必须验证新闻内容是否明确提到该单位名称：**

| 用户输入 | 新闻内容 | 匹配结果 | 说明 |
|---------|---------|:--------:|------|
| 绿地建设 | "上海绿地建设集团破产审查" | ✅ 匹配 | 全称匹配 |
| 绿地建设 | "绿地控股发布财报" | ⚠️ 需确认 | 母公司，需标注 |
| 中建三局 | "中建三局某项目发生事故" | ✅ 匹配 | 直接匹配 |
| 中建 | "中建八局中标项目" | ❌ 不匹配 | 不同单位 |
| 某设计院 | "某设计院裁员" | ❌ 不匹配 | 未明确单位 |

## 标准输出格式

**⚠️ 使用 `[平台名称](链接)` 格式隐藏URL**

```markdown

---
```
