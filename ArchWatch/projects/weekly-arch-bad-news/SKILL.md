# 一周建筑烂新闻

---

# ⚠️ 建筑行业周报必须严格执行：严格≤7天

## 时效性计算方法（强制执行）

```
1. 获取新闻发布日期
2. 计算天数差 = 触发日期 - 新闻发布日期
3. 判断：
   - 天数差 ≤ 7 → 可以收录
   - 天数差 > 7 → 坚决排除，禁止收录
```

**示例（触发日期为2026年3月2日）：**
- 3月2日发布 → 天数差=0 → ✅ 可以收录
- 3月1日发布 → 天数差=1 → ✅ 可以收录
- 2月23日发布 → 天数差=7 → ✅ 可以收录（边界）
- 2月22日发布 → 天数差=8 → ❌ 坚决排除
- 2月20日发布 → 天数差=10 → ❌ 坚决排除
- 1月28日发布 → 天数差=33 → ❌ 坚决排除

**警告：任何天数差>7的新闻，不管内容如何，坚决禁止收录！**

# 功能1：建筑行业周报

## 收录标准（必须同时满足，缺一不可）

**标准1：时效性（必须严格≤7天）**
- 天数差 = 触发日期 - 新闻发布日期
- 必须天数差 ≤ 7 才能收录
- 天数差 > 7 坚决排除，禁止收录

**标准2：行业相关性**
- 必须是建筑行业：设计院、施工单位、开发商、建筑类高校
- 禁止收录：互联网公司（程序员、腾讯、阿里、字节等）
- 禁止收录：游戏公司（米哈游、网易游戏等）
- 禁止收录：金融行业、制造业等非建筑行业

## 操作步骤

### 步骤1：搜索新闻
使用 web_search 搜索以下关键词（必须加"近一周"）：
- `设计院 裁员 近一周`
- `设计院 降薪 近一周`
- `建筑企业 破产 近一周`
- `施工单位 讨薪 近一周`
- `建筑专业 撤销 近一周`

### 步骤2：严格过滤（必须逐条检查）

**对每条新闻，必须执行以下检查，不满足条件的立即丢弃：**

**检查A：时效性（第一关，必须先检查）**
```
1. 读取新闻发布日期
2. 计算天数差 = 触发日期 - 新闻发布日期
3. 判断：
   - 天数差 ≤ 7 → 通过，继续检查B
   - 天数差 > 7 → 立即丢弃，禁止收录，不再检查
```

**检查B：行业相关性（第二关）**
```
1. 检查新闻主体是否属于建筑行业
2. 判断：
   - 属于 → 收录该新闻
   - 不属于 → 立即丢弃，禁止收录
```

**错误示例（不要这样做）：**
- ❌ 2月22日发布的新闻（8天前）→ 必须排除
- ❌ 程序员猝死新闻 → 不是建筑行业，必须排除
- ❌ 米哈游员工猝死新闻 → 是游戏公司，必须排除

### 步骤3：输出新闻列表

**标题日期规则：**
- 结束日期 = 触发日期（今天）
- 起始日期 = 触发日期往前数7天
- 示例：触发日期3月2日 → 起始日期2月23日 → 标题"2月23日 - 3月2日"

按以下格式输出：

```markdown
# 一周建筑烂新闻 | MM月DD日 - MM月DD日

> 📊 本周共收录 **N** 条行业负面新闻
```

# 功能2：单位负面新闻查询

2. 搜索关键词（不限时间）：
   - `{单位名称} 裁员`
   - `{单位名称} 降薪`
   - `{单位名称} 欠薪`
   - `{单位名称} 破产`
   - `{单位名称} 纠纷`
   - `{单位名称} 维权`

3. 验证：新闻必须明确提到该单位名称
